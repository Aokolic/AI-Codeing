## 一、后端 API 模块

### 1.1 案例模块 `/api/v1/cases`

**功能说明**：提供舆论案例的增删改查、全文搜索和多维过滤能力，是平台的核心数据组织单元。

| 方法 | 路径 | 功能 | 关键参数 |
|------|------|------|---------|
| `GET` | `/cases` | 分页获取案例列表 | `page`/`page_size`/`status`/`tag_id`/`sort`(hotness\|created_at) |
| `GET` | `/cases/search` | 全文搜索案例 | `q`(1-200字符)，`limit`(默认10，上限50) |
| `GET` | `/cases/{case_id}` | 获取案例详情 | — |
| `POST` | `/cases` | 创建新案例 | body: `title`/`description`/可选`tag_ids` |
| `PATCH` | `/cases/{case_id}` | 更新案例信息 | body: 任意字段(均可选) |

### 1.3 可信度评估模块 `/api/v1/events/{id}/credibility`

**功能说明**：对指定事件节点的所有关联来源执行三维可信度计算（权威性/时效性/交叉验证），返回综合评分、等级和详细解释。结果写入数据库缓存，不重复计算。

| 方法 | 路径 | 功能 |
|------|------|------|
| `GET` | `/events/{event_id}/credibility` | 获取/计算可信度评估 |

**返回结构**：
```
CredibilityOverall
├── level: high | medium | low | unverified
├── total_score: 0-100
├── authority_score, timeliness_score, cross_verify_score
├── source_count: int
├── has_conflict: bool
├── conflict_sources: list[str]
└── scoring_explanation: str（中文自然语言解释）
```

## 二、数据模型层

### 2.1 Case（舆论案例）

**业务含义**：某一持续关注的社会事件或信息真伪存疑的话题，聚合来自多个媒体的相关报道。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | UUID String | 主键 |
| `title` | String(200) | 案例标题，不可为空 |
| `description` | Text | 案例描述，可为空 |
| `status` | Enum | `active`(活跃) / `observing`(观察) / `closed`(关闭) |
| `hotness_score` | Float | 综合热度分（0-100） |
| `last_event_at` | DateTime(UTC) | 最近一次关联事件的时间 |
| `created_at` / `updated_at` | DateTime(UTC) | 创建/更新时间 |

关联：`Case` 1→N `EventNode`，`Case` M↔N `Tag`（via `CaseTag`）

---

### 2.2 EventNode（事件节点）

**业务含义**：案例时间线上的单个事件进展，对应一篇新闻文章或一条报道，存储 NLP 向量用于语义匹配。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | UUID String | 主键 |
| `case_id` | FK → cases | 所属案例（CASCADE 删除） |
| `title` | String(300) | 事件标题 |
| `summary` | Text | 事件摘要 |
| `event_time` | DateTime(UTC) | 事件发生时间 |
| `location` | String(200) | 事件地点（可空） |
| `embedding_json` | Text | 768 维语义向量（JSON 序列化，可空） |

索引：`idx_event_case_time(case_id, event_time)` · `idx_event_time(event_time)`

---

### 2.3 Source（来源）

**业务含义**：某篇报道的原始来源记录，包含来源的类型和可信度元数据，用于可信度评估。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | UUID String | 主键 |
| `name` | String(200) | 来源名称 |
| `source_type` | Enum | `government` / `mainstream_media` / `academic` / `local_media` / `social_media` / `unknown` |
| `url` | String(2048) | 原文链接（URL 去重依据） |
| `reputation_score` | Float | 声誉评分（默认 50.0） |
| `has_false_history` | Boolean | 是否有虚假报道历史（影响权威性评分） |
| `collected_at` | DateTime(UTC) | 采集时间（用于时效性计算） |
| `data_feed_id` | FK → data_feeds | 采集来源的 Feed（可空，SET NULL） |

关联：`Source` M↔N `EventNode`（via `EventNodeSource`，含 `raw_content` 原始内容和 `relevance_score` 相关性分）

---

### 2.4 DataFeed（数据源）

**业务含义**：配置一个 RSS/JSON 采集端点，记录采集健康状态。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | UUID String | 主键 |
| `name` | String(200) | 数据源名称 |
| `feed_type` | Enum | `rss` / `api` / `scraper` |
| `url` | String(2048) | 采集 URL |
| `parse_config` | JSON | 解析配置（scraper 用，null=自动） |
| `schedule_cron` | String(50) | 定时表达式（默认 `*/30 * * * *`） |
| `is_builtin` | Boolean | 是否为内置源（影响删除/编辑权限） |
| `status` | Enum | `normal` / `warning` / `offline` |
| `consecutive_failures` | Integer | 连续失败次数（≥3 → warning） |
| `last_collected_at` | DateTime(UTC) | 最近采集成功时间 |

---

### 2.5 CredibilityAssessment（可信度评估）

**业务含义**：EventNode 的三维可信度评分结果，数据库级缓存，避免重复计算。

| 字段 | 类型 | 说明 |
|------|------|------|
| `event_node_id` | FK → event_nodes | 关联事件（唯一约束） |
| `level` | Enum | `high` / `medium` / `low` / `unverified` |
| `total_score` | Float | 综合分（0-100） |
| `authority_score` | Float | 权威性分 |
| `timeliness_score` | Float | 时效性分 |
| `cross_verify_score` | Float | 交叉验证分 |
| `source_count` | Integer | 来源总数 |
| `has_conflict` | Boolean | 是否存在来源冲突 |
| `conflict_sources_json` | Text | 冲突来源名称（JSON 数组） |

## 三、业务服务层

### 3.1 collector.py — 采集与分类引擎

**功能说明**：从 RSS/JSON 端点拉取文章，完成去重、NLP 分类、事件节点创建、来源关联的全自动流程。

**核心流程**：
```
fetch_url → 协议路由(RSS/JSON) → 文章解析
    → URL 去重（跳过已收录来源）
    → _classify_to_case()：双层 NLP 找最匹配案例
        ├─ Layer 1: jieba 实体提取 + Jaccard 重叠度
        └─ Layer 2: sentence-transformers 余弦相似度
    → find_matching_event()：±48h 窗口内事件去重（阈值 0.82）
    → 创建/关联 EventNode + Source
    → 更新 feed.last_collected_at
```

**测试条件**：

| 测试场景 | 预期结果 | 测试文件 |
|---------|---------|---------|
| 相同主题文章 | 归入同一 Case | `test_classify_case.py` |
| 无关文章 | 创建新 Case | `test_classify_case.py` |
| 语义相似度 ≥ 0.45 | 匹配成功（路径B） | `test_classify_case.py` |
| 实体+语义联合 | 路径A 匹配成功 | `test_classify_case.py` |
| gov.cn URL | 检测为 `government` | `unit/test_collector.py` |
| weibo.com URL | 检测为 `social_media` | `unit/test_collector.py` |
| 合法 RSS XML | 解析出正确 title | `unit/test_collector.py` |
| 空 RSS 频道 | 返回 `[]` | `unit/test_collector.py` |

---

### 3.2 entity_extractor.py — 实体提取

**功能说明**：基于 jieba 词性标注提取文本中的命名实体（人名/地名/机构名/专有名词）及有效名词，用于第一层分类的 Jaccard 相似度计算。

**提取规则**：
- 词性 `nr`(人名) / `ns`(地名) / `nt`(机构名) / `nz`(专有名词)：长度 ≥ 2 即收录
- 词性 `n`(普通名词)：长度 ≥ 2 且不在 30+ 停用词列表中方收录
- 返回 `set[str]`，用于 Jaccard 系数：$J(A,B) = |A∩B| / |A∪B|$

**测试条件**：

| 测试场景 | 预期结果 | 测试文件 |
|---------|---------|---------|
| 含地名文本 | 提取出"伊朗"/"北京"等 | `test_entity_extractor.py` |
| 含人名文本 | 提取出人名 | `test_entity_extractor.py` |
| 含机构名文本 | 提取出机构名 | `test_entity_extractor.py` |
| 空字符串 | 返回空集合 `set()` | `test_entity_extractor.py` |
| 完全相同集合 | `overlap = 1.0` | `test_entity_extractor.py` |
| 完全不同集合 | `overlap = 0.0` | `test_entity_extractor.py` |
| 部分重叠集合 | 正确 Jaccard 值 | `test_entity_extractor.py` |
| 双空集合 | `overlap = 0.0`（不除零）| `test_entity_extractor.py` |

---

### 3.3 credibility.py — 可信度评估引擎

**功能说明**：对 EventNode 的所有关联来源执行三维评分，计算权威性/时效性/交叉验证分数并加权合并，检测来源冲突，生成中文解释文本。

**评分公式**：
$$S_{total} = S_{authority} \times 0.4 + S_{timeliness} \times 0.2 + S_{cross} \times 0.4$$

**等级映射**：≥75 → high | 50-74 → medium | 25-49 → low | <25 → unverified

**测试条件**：（同 1.3 节已列出）

### 3.5 hotness.py — 热度计算

**功能说明**：综合更新频率、来源多样性、案例生命周期状态计算热度分数，用于首页排序。

**公式**：$H = freq \times 0.4 + sources \times 0.3 + status \times 0.3$，上限 100

**测试条件**：

| 来源数增加 | 热度更高 | `unit/test_hotness.py` |
| 满分场景 | `score ≤ 100` | `unit/test_hotness.py` |

### 3.6 case_lifecycle.py — 生命周期状态机

**功能说明**：自动管理案例的三态转换（active/observing/closed），由调度器每天凌晨 3 点批量执行。

**转换规则**：
- `active` → `observing`：30 天无新事件
- `observing` → `closed`：90 天无新事件
- 任意状态 + 新事件 → 立即恢复 `active`

### 5.2 `Timeline.vue` — 事件时间线

**功能说明**：纯容器组件，渲染事件列表，冒泡点击事件。

| Props | 类型 | 说明 |
|-------|------|------|

**行为**：加载中 → `NSpin`；无事件 → `NEmpty`；正常 → 逐条渲染 `EventCard`；向上 emit `event-click(eventId)`

**测试条件**：

| 空数组 | html 含"暂无"/"empty"相关文字 |
| 点击 EventCard | emit `event-click` |
| 时间顺序 | 第一条出现在第二条之前 |
