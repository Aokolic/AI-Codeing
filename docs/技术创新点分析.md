# 「后真相时代」信息甄别平台 — 技术创新点分析

---

## 创新点一：双层融合 NLP 智能事件归类引擎

### 创新思路

传统新闻聚合系统通常采用单一的关键词匹配或纯语义模型进行文章分类，前者准确率低、后者计算成本高。本项目提出了**实体重叠 + 语义相似度的双层渐进式分类策略**，在轻量级词法分析与重量级深度语义模型之间取得了精度与性能的平衡，并支持模型不可用时的优雅降级。

### 实现过程

**第一层（轻量级）：jieba 命名实体提取 + Jaccard 重叠度**

基于 jieba 词性标注（POS Tagging），提取文本中的人名（nr）、地名（ns）、机构名（nt）、专有名词（nz）及有效名词（≥2 字且非停用词），构建实体集合。通过 Jaccard 系数衡量两篇文本的实体重叠程度：

$$J(A, B) = \frac{|A \cap B|}{|A \cup B|}$$

此层无需加载深度学习模型，单次调用耗时 < 5ms，适合快速筛选。

**第二层（精确级）：sentence-transformers 语义编码 + 余弦相似度**

采用 `shibing624/text2vec-base-chinese` 模型（768 维向量），对文本进行 L2 归一化后编码，通过余弦相似度衡量语义相关性。模型惰性加载（首次调用时初始化），全局单例避免重复加载。

**融合决策矩阵**

系统设计了三条匹配路径，形成鲁棒的融合决策：

| 路径 | 条件 | 含义 |
|------|------|------|
| 路径 A | 实体重叠 ≥ 0.3 **且** 语义 ≥ 0.35 | 双重验证：实体匹配+语义辅助确认 |
| 路径 B | 语义 ≥ 0.45 | 纯语义高置信（实体不重叠但表达同一事件） |
| 路径 C | 模型不可用 **且** 实体重叠 ≥ 0.3 | 优雅降级：NLP 模型故障时仍可工作 |

当存在多个候选案例时，系统通过 `combined = overlap + semantic_score` 综合排序选出最优匹配。

**性能缓存机制**

在单次采集任务中，每篇新文章需与所有已有案例对比，产生 O(文章数 × 案例数) 次模型调用。通过引入 `_embedding_cache`（向量缓存）和 `_entity_cache`（实体缓存），同一文本的编码结果在任务内复用，任务结束后统一释放，避免内存泄漏。

### 效果分析

| 指标 | 优化前 | 优化后 |
|------|--------|--------|
| 分类策略 | 纯语义余弦（单阈值 0.55） | 双层融合（实体 0.3 + 语义 0.35/0.45） |
| 模型故障容错 | 分类完全失效 | 路径 C 降级为实体匹配，系统正常运行 |
| 单次采集耗时（8 源 × 20 篇 × 50 案例） | ~120s（重复推理） | ~15s（缓存命中后跳过） |
| 分类准确性 | 同一事件分散到多个案例 | 实体+语义双验证，相关报道正确聚合 |

## 创新点二：三维可信度量化评估模型

### 创新思路

信息真实性判断不能仅依赖单一指标。本项目借鉴新闻核实（Fact-Checking）领域的三角验证理论，提出**权威性 × 时效性 × 交叉验证**三维量化评估模型，将主观的"可信/不可信"转化为 0-100 的可量化评分，并输出结构化的中文解释文本，提升评估的透明度和可解释性。

### 实现过程

**评分公式**

$$S_{total} = S_{authority} \times 0.4 + S_{timeliness} \times 0.2 + S_{cross} \times 0.4$$

**维度一：来源权威性（权重 40%）**

建立来源类型到基准分的映射体系：

| 来源类型 | 基准分 | 示例 |
|---------|--------|------|
| 政府公告（government） | 90 | 国务院、官方公告 |
| 学术来源（academic） | 85 | 研究机构、学术论文 |
| 主流媒体（mainstream_media） | 80 | BBC、NYT、DW |
| 地方媒体（local_media） | 60 | 区域性新闻媒体 |
| 社交媒体（social_media） | 30 | 微博、Twitter |
| 未知来源（unknown） | 10 | 无法识别 |

通过来源名称和 URL 的关键词匹配自动检测来源类型（覆盖中英文关键词 20+），并对有虚假报道历史的来源施加 50% 惩罚因子。

**维度二：时效性衰减（权重 20%）**

采用线性衰减模型，反映新闻信息随时间推移的可靠性递减：

$$S_{timeliness}(h) = \max\left(0,\ 100 - \frac{h}{24} \times 5\right)$$

其中 $h$ 为事件发生至今的小时数。每天衰减 5 分，20 天后时效性归零。这一设计体现了新闻核实中"时间越近越可信"的基本原则。

**维度三：交叉验证（权重 40%）**

多源异质性验证是信息核实的核心方法。系统统计报道同一事件的独立来源数量及来源类型多样性：

| 来源数 | 评分 | 条件 |
|--------|------|------|
| 1 | 20 | 孤证 |
| 2 | 50 | 双源印证 |
| 3 | 75 | 三方验证 |
| 4 | 90 | 四方验证 |
| ≥5 且 ≥2 种类型 | 100 | 多源异质性满分 |

同时实现了**来源冲突检测**：当 ≥2 个高权威来源（权威分 ≥75）共同报道但内容存在分歧时，系统自动生成冲突预警。

**可解释性输出**

每次评估不仅输出数值分数，还生成结构化的中文解释（`scoring_explanation`），包括各维度得分、等级判定和预警信息，使非专业用户也能理解评分依据。

### 效果分析

| 评估维度 | 设计效果 |
|---------|---------|
| 区分度 | 4 个等级（high ≥75 / medium 50-74 / low 25-49 / unverified <25）有效区分不同可信度 |
| 可解释性 | 每个评分附带自然语言解释，不是"黑盒"分数 |
| 冲突检测 | 自动标记高权威来源间的矛盾报道，提示读者批判性阅读 |
| 动态衰减 | 时效性维度随时间自动更新，无需人工干预 |

## 创新点三：跨平台多协议聚合采集引擎

### 创新思路

国际中文媒体的数据接口格式各异（RSS/XML、JSON API、HTML 页面），传统爬虫需要为每个站点编写专用解析器。本系统设计了**协议自适应采集引擎**，通过 URL 特征自动路由到对应解析器，并实现了多字段名兼容和双重日期解析策略，一套代码适配 8 个跨国/跨地区媒体源。

### 实现过程

**三协议自适应路由**

```
fetch_url → URL后缀判断
              ├── .json → _parse_json_feed (NHK等JSON API)
              ├── .xml/.rss → feedparser (标准RSS)
              └── 默认 → feedparser (自动检测)
```

**JSON Feed 自适应解析**

针对 NHK 等非标准 JSON 接口，实现了多层结构探测：支持 `[]`（直接数组）、`{"data":[]}`、`{"channel":{"item":[]}}` 等多种 JSON 结构；标题字段兼容 `title`/`headline`，摘要字段兼容 `summary`/`description`/`lead`/`snippet`；日期解析采用 RFC2822 → ISO8601 的双重回退策略。

**智能来源类型检测**

通过来源名称和 URL 的关键词匹配（中英文 20+ 个关键词），自动识别来源类型（政府/主流/学术/地方/社交/未知），无需人工标注，直接影响可信度评估中的权威性维度。

**健康监控与自愈**

每次采集更新 `consecutive_failures` 计数器，连续失败 ≥3 次自动标记 `warning` 状态。采集成功后自动清零，实现自愈。

### 效果分析

| 指标 | 效果 |
|------|------|
| 源覆盖 | 8 个国际中文源覆盖 7 个国家/地区（英/德/法/美/韩/日/新加坡+台湾） |
| 协议兼容 | RSS/XML + JSON API 双协议，一套代码适配 |
| 容错能力 | 网络异常、格式变更不影响其他源采集，健康监控自动预警 |
| 零配置 | 首次启动自动创建所有内置源并开始采集 |

## 创新点四：全自动化 AI Agent 工作流管线

### 创新思路

传统信息核实流程高度依赖人工：记者手动收集文章 → 比对来源 → 编写核实报告。本系统将整个流程设计为**全自动无人值守管线**，从数据采集、NLP 分类、可信度评估到案例生命周期管理，全部由 AI Agent 自动完成，实现"一键启动，持续运行"。

### 实现过程

**自动化工作流链路**

```
[APScheduler 定时触发] → [多源并行采集] → [URL 去重]
         ↓
[jieba 实体提取] → [实体重叠匹配] → [语义相似度验证]
         ↓
[案例归类/新建] → [事件节点去重(±48h, 余弦≥0.82)]
         ↓
[来源记录 + 关联写入] → [可信度自动评估]
         ↓
[案例生命周期状态机自动转换]
```

**任务调度**

| 定时任务 | 频率 | 功能 |
|---------|------|------|
| 全量采集 | 每 30 分钟 | 自动遍历所有活跃数据源 |
| 生命周期转换 | 每天凌晨 3 点 | 批量检查案例状态，自动转换 |

配置 `misfire_grace_time=300s`，短暂宕机恢复后不遗漏任务。

**案例生命周期有限状态机**

```
              新事件产生
    ┌──────────────────────────┐
    ↓                          │
 [active] ──30天无更新──→ [observing] ──90天无更新──→ [closed]
    ↑                          ↑
    └──── 新事件自动激活 ────────┘
```

三种状态自动转换，无需人工干预。任何状态下收到关联新事件，立即回到 `active`。

**零配置启动**

应用启动时自动执行：建表 → 种子数据（幂等） → 启动调度器；关闭时优雅停止调度器。首次运行到产出分析结果，中间无需任何人工操作。

### 效果分析

| 指标 | 效果 |
|------|------|
| 人工干预 | 零次（从启动到产出完全自动） |
| 采集频率 | 每 30 分钟全量更新，信息延迟 < 30 分钟 |
| 生命周期管理 | 自动休眠不活跃案例，节省计算资源 |
| 系统鲁棒性 | 宕机重启自恢复，单源故障不影响全局 |

## 创新点五：性能与体验优化 — 量化对比分析

### 创新思路

信息甄别系统的用户价值取决于"及时"和"准确"。本项目针对 NLP 推理性能瓶颈和前后端交互体验，进行了多层次的量化优化。

### 实现过程与量化对比

**优化一：NLP 推理缓存**

| 场景 | 优化前 | 优化后 | 提升 |
|------|--------|--------|------|
| 20 篇文章 × 50 案例对比 | 1000 次模型推理 | ~70 次（去重后） | **~14× 加速** |
| 单次全量采集（8 源） | ~120 秒 | ~15 秒 | **~8× 加速** |

**优化二：模型惰性加载**

sentence-transformers 模型（~400MB）采用惰性加载：首次 NLP 调用时初始化，后续全局复用。避免启动时 10+ 秒的模型加载阻塞。

**优化三：前端同步采集 + 即时反馈**

| 指标 | 优化前 | 优化后 |
|------|--------|--------|
| 采集结果反馈 | 异步后台执行，用户无感知 | 同步等待完成，按钮 loading 状态实时显示 |
| 数据刷新 | 需手动刷新页面 | 采集完成后自动调用 `loadFeeds()` 刷新列表 |
| 时间显示 | UTC 时间（与本地差 8 小时） | 自动转换为浏览器本地时区 |
| 采集 API 超时 | 默认 15 秒 | 120 秒（适应 NLP 处理时间） |

**优化四：事件级去重**

通过 ±48 小时时间窗口 + 0.82 余弦相似度阈值进行事件节点级去重，避免同一事件被不同来源重复创建为多个时间线节点。

### 效果分析

整体系统响应链路：

```
用户点击"立即采集" → 按钮 loading(0s) → RSS 下载(~2s) → NLP 分类(~5s, 缓存)
    → 可信度评分(~1s) → DB 写入(~1s) → 返回完成 → 列表自动刷新(~0.5s)
    → 总耗时：~10 秒，用户全程可见进度
```

| 创新维度 | 本项目创新点 | 关键技术 |
|---------|------------|---------|
| AI Agent 自动化流程 | 全自动采集→分类→评估→生命周期管线 | APScheduler + 有限状态机 |
| 跨平台/API 集成 | 8 国中文媒体多协议聚合 | RSS/JSON 自适应 + 智能类型检测 |
| 个性化用户体验 | 三维可信度 + 可解释性评分 | 加权评分模型 + 冲突检测 + 中文解释 |
| 性能与体验优化 | 双层 NLP 缓存 + 同步采集反馈 | 推理缓存 ~14× 加速 + 即时 UI 刷新 |
| 核心算法创新 | 实体重叠 + 语义相似度融合分类 | jieba + sentence-transformers 双层决策矩阵 |
